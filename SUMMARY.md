### 3. Оптимизирован расчет flaky

**Критичная проблема:** 
- Медленная обработка в Python вместо SQL в `compute_flaky()`
- Чтение одних и тех же колонок внутри цикла (N+1 queries)

**Решение:**
- Полностью переписана функция с использованием CTE
- Агрегация данных через `GROUP BY` и `JOIN` — один запрос вместо цикла
- Добавлены индексы на `external_id`, `test_case_id` для ускорения запросов


### 4. Приведена работа с БД к единообразию

**Схема данных:**
- Унифицирована переменная `external_id` везде вместо смешивания `test_external_id`
